# X Ops v2

## 核心策略：两个阶段，两种打法

X 增长的本质是**信任积累**，不同阶段信任来源不同：

| 阶段 | 粉丝数 | 信任来源 | 核心动作 |
|------|--------|----------|----------|
| **Tier 1: 冷启动** | 0-500 | 别人的账号 | 在大号帖子下留优质评论，借流量建声誉 |
| **Tier 2: 建立权威** | 500-2000 | 自己的内容 | 在 niche 内做最有用的声音，建内容壁垒 |
| **Tier 3: 规模化** | 2000+ | 账号本身 | 确保评论和帖子质量，系统化输出 |

**两个阶段的最大误区：**
- Tier 1 账号花太多时间写帖子 → 没人看，浪费精力
- Tier 2 账号继续乱评论不相关的帖子 → 稀释 niche 定位

**Tier 1 冷启动加速器：X Communities**

X Communities 是 Tier 1 最被低估的工具，能绕过零粉丝问题：
- "Build in Public" community 有 180K+ 成员，直接获得精准受众曝光
- 发到 community 的帖子对所有成员可见，不受你粉丝数限制
- 建议：Tier 1 阶段，**每条帖子同时发到 1-2 个相关 community**
- 操作：发帖时选择 "Post to a community"，找到对应 niche 的 community

## ⚪ Module 0: Profile（账号画像）

**每次换 handle 必须先跑。其他模块的输出质量依赖这个。**

### 执行流程

```
Step 1: 基本信息（浏览器）
├── 导航到 x.com/@{handle}
├── 抓取：Bio、固定推、粉丝数、关注数、加入日期
├── 判断阶段：Tier 1 (<500) / Tier 2 (500-2000) / Tier 3 (2000+)
├── 检查是否有 X Premium（蓝色 ✓ 标记）
│   → 未开通时提醒：Premium 账号在回复和搜索中有约 30% 可见度加成
│   → 对 Tier 1 尤为重要，是最低成本的曝光提升手段
└── 抓取最近 20 条原创帖（跳过转发）

Step 2: 内容支柱
├── 分析 20 条帖子的主题分布
├── 找出 3-5 个核心话题领域
└── 记录每个支柱的代表帖

Step 3: 声音指纹
├── 句子长度：短(<80字符) / 中 / 长
├── 语气：干货型 / 故事型 / 观点型 / 互动型
├── Emoji 频率：不用 / 偶尔 / 常用
├── 标志性词汇或句式（5-10个）
└── 这个账号绝不会说的表达

Step 4: 独家资产
├── 这个人有什么别人没有的：经历 / 数据 / 职业背景 / 内部观察
└── 一句话总结：这个账号凭什么被关注？

Step 5: 世界观提取
├── 通读所有帖子，找出反复出现的核心信念或立场
├── 用一句话总结：这个账号认为"[某个领域]的本质是[X]，大多数人误解了[Y]"
├── 示例：@MeredithCheng22 的世界观 = "AI产品失败不是技术问题，是founder在满足自己的想象而不是用户的现实"
└── 这个世界观之后用于审核每条帖子：这条帖子在强化世界观吗？

Step 6: 高效内容分析（仅限已有帖子的账号）
├── 互动率最高的 3-5 条帖子 → 共同规律
└── 互动率最低的 2-3 条 → 失败原因
```

## 互动网络（三层名单）
X 增长不是单向的（我去评论别人），而是需要建立一个互相关注、互相 engage 的小圈子。
维护三层名单，每天按顺序花 15-20 分钟：

**层 1 — 标杆大号（3-5个）**：粉丝 50K+，你想被他们圈子的人看到
→ 操作：开通帖子通知，争取前 10 分钟内评论，不求 follow back，求曝光

**层 2 — 同体量账号（10-15个）**：粉丝数与你相近（±50%），相同 niche
→ 操作：互相 follow，互相评论对方帖子，形成互相扩散的增长圈
→ 这层是核心：follow-back 概率最高，互相 amplify 效果最好

**层 3 — 潜力新人（5-10个）**：粉丝 <你的 50%，但内容质量高
→ 操作：主动 follow 并评论，建立早期关系（他们长大后会记得你）

- 同体量账号（互扶圈）: {@账号1, @账号2, @账号3}
- 潜力新人: {@账号1, @账号2}

**依赖 Profile。每日早晨运行一次，输出今日行动包（Signal Pack）。**

每日监控分三层，按顺序执行，最终打包成一个审批单：

| 层级 | 信号来源 | 目的 |
|------|----------|------|
| **Layer A** | 外部新闻（web_search） | 找行业动态，判断有无可蹭热点 |
| **Layer B** | X 平台热点 + Niche 圈子 | 找正在发酵的帖子，判断评论/QT 机会 |
| **Layer C** | 账号近期表现 | 找出什么在 work，调整今日重点 |

### 大热点 vs Niche 小热点：哪个更容易突破？

对 Tier 1/2 账号来说，**niche 内的小圈子讨论往往比全平台大热点更有效**：

| | 全平台大热点 | Niche 小热点 |
|--|------------|------------|
| 竞争程度 | 极高（几万条帖子） | 低（几十条） |
| 相关受众密度 | 低（大量无关用户） | 高（都是你的目标受众） |
| 视角空间 | 几乎被占满 | 往往有独特角度 |
| 时效窗口 | 2-4 小时 | 更长（niche 讨论持续 1-3 天） |
| 适合账号阶段 | Tier 3+（大号才有话语权） | Tier 1/2 最有效 |

**结论**：Tier 1/2 账号优先找 niche 小热点（某个 KOL 发了引发讨论的帖子、niche 内的争议性观点），而不是追全平台大热点。

### X 算法对热点的影响

- **热点时间窗口极短**：大多数趋势在 X 上只热 2-6 小时，超过 8 小时基本没意义
- **QT 比普通帖优先分发**：QT 热帖时，X 会把你的帖子推给看过原帖的人
- **视角套利**：前 10 条帖子占据了"这个话题的主要观点"，第 50 条帖子很难被看到
- **关键词匹配**：帖子里出现热门关键词会被算法额外推送给关注该话题的人

### 执行流程

#### Layer A: 外部新闻扫描（web_search）

```
Step A1: 基于 Profile 内容支柱生成搜索词
├── 读取 Profile 中的"内容支柱"（3-5个）
├── 每个支柱生成 1 个搜索词（英文，加 "news today" 或 "2026"）
│   示例：支柱是"AI 工具" → 搜索 "AI agent news today"
│   示例：支柱是"产品增长" → 搜索 "product-led growth news 2026"
│   示例：支柱是"创业融资" → 搜索 "startup funding news today"
├── 加 1-2 个行业层面的必搜词（覆盖账号所在大领域）
└── 时效过滤：只看 24h 内的结果

Step A2: 热点筛选
从搜索结果中筛选与账号相关的：
- 大厂动态（直接影响账号所在 niche 的）
- 行业融资/收购
- 新产品/功能发布
- 有观点分歧的热门话题
→ 不相关的直接跳过，不要凑数量

Step A3: 可蹭性评估
每个热点判断：
- 账号内容支柱匹配度（高/中/低）
- 账号有没有独特角度（有 → 蹭；没有 → 跳过，不强行）
- 蹭法：正面支持 / 反向观点 / 借势科普 / QT 原链
→ 匹配度低 + 无独特角度 = 直接跳过
```

#### Layer B: X 平台 + Niche 圈子（浏览器）

```
Step B1: X 平台热点
├── 导航到 x.com/explore/tabs/for-you
├── 抓取 "Today's News" 区块：标题 + 帖子数量
└── 记录时效（距首帖多久了？）

Step B2: Niche 小热点扫描 — 优先级高于 B1
├── 浏览 Profile 互动网络"层 1 + 层 2"名单中的账号
├── 查看 12h 内有异常高互动的帖子（评论数突然增多）
├── 识别 niche 内正在发酵的讨论或争议
└── 找到"有人说了 X，引发了不同看法"的话题 → 最好的 niche 热点
   → Niche 热点信号：同领域多个账号在评论同一条帖子，或同话题下出现多个新帖
```

#### Layer C: 账号近期表现（浏览器）

```
Step C1: 拉取最近 10 条原创帖数据
├── 导航到 x.com/@{handle}
├── 跳过转发，只看原创帖
├── 记录每条：发布时间 / views / likes / 评论数 / 书签数
└── 计算近期平均互动率（likes ÷ views）

Step C2: 高效内容识别
├── 互动率 > 均值 1.5x 的帖子 → 分析共同特征
│   问：Hook 类型？内容类型（数据/故事/观点）？发帖时间？
└── 记录：今日应强化的方向

Step C3: 低效内容识别
├── 互动率 < 均值 50% 的帖子 → 分析失败原因
│   问：Hook 太弱？语气太营销？内容太泛？
└── 记录：今日应避免的模式

→ 如果账号新建（<5条原创帖），跳过 Layer C，直接进内容队列检查
```

#### 内容队列检查

```
检查本次对话中是否有待开发的选题（用户之前提过的话题/ideas）：
├── 有 → 列为今日优先候选，排在热点之前
├── 无 → 从 Layer A + B 的信号中提炼 2-3 个选题
│        按"账号独特角度 + 时效"排序
└── 队列选题 + 今日热点 → 合并作为今日内容来源
```

### 机会评估（四维度）

每个热点按以下维度评分：

| 维度 | 高分 | 低分 |
|------|------|------|
| **支柱匹配** | 直接命中账号核心支柱 | 与任何支柱无关 |
| **视角独特** | 账号能说一个别人没说的角度 | 已有 50+ 帖子说同样的话 |
| **时效性** | <4 小时（QT 最有效）/ <8 小时（发帖还行） | >12 小时 → 跳过 |
| **可信度** | 账号有亲身经历/数据支撑这个观点 | 只能说通用观点 |

**行动决策树：**
```
支柱匹配 ✅ + 时效 <4h + 视角独特 ✅ → 立即 QT + 原创观点
支柱匹配 ✅ + 时效 <4h + 视角一般 → 评论热帖（不必发新帖）
支柱匹配 ✅ + 时效 4-12h + 有独特角度 → 可以发借势帖
支柱匹配 ✅ + 时效 >12h → 跳过，这个话题做原创选题
支柱不匹配 → 跳过，无论多热
```

## 🟡 Module 2: Comment（评论）

**Tier 1 账号最重要的模块。评论是你在别人流量里建立声誉的唯一方式。**

### X 算法对评论的影响

**这是找到高质量回复目标的底层逻辑：**

1. **评论排序机制**：X 优先展示"能引发互动的评论"——即评论本身有 likes 和 replies 的。你的评论不只是给原帖作者看的，是给所有看这条帖子的人看的。

2. **Early comment 优势**：帖子发出后 30-60 分钟内的评论，在帖子生命周期内持续曝光；2小时后进入的评论能见度下降 70%+。

3. **账号权重**：粉丝越多的账号评论排名越靠前——这意味着 Tier 1 账号需要用**评论质量**弥补**账号权重**不足。

4. **目标帖子的选择**：评论目标帖的流量决定了你评论的潜在曝光上限。帖子 views=10K 的帖子，你的评论最多被几百人看到；帖子 views=100K 的帖子，你的评论可能被几千人看到。

### Tier 1 (0-500) 评论策略：最大化每条评论的 ROI

**目标：每条评论让 5-10 个人点进你的主页**

**选帖公式（Tier 1 专用）：**

```
理想目标帖特征：
✅ 作者粉丝：10K-100K（中型 KOL，帖子有流量但评论区不过载）
✅ 帖子 views：5K-50K（有流量，你的评论不会被淹没）
✅ 现有评论数：< 20 条（黄金窗口：前 20 条评论最显眼）
✅ 发布时间：< 90 分钟（越早越好）
✅ 话题：直接落在账号内容支柱内
✅ 帖子类型：有明确论点的（观点帖、数据帖）而不是情绪帖

⚠️ 次优但可接受：
- 评论数 20-40，发布 1.5-3 小时
- 作者粉丝 100K-500K（评论可见度低，但偶尔蹭大号也值得）

❌ 跳过：
- 评论 > 50 条（你的评论会消失在海里）
- 发布 > 4 小时
- views < 1K（没流量）
- 话题与账号无关（浪费）
- 政治/争议话题
```

**找目标帖的操作流程（浏览器）：**

```
Step 1: Following tab 优先（算法显示你关注的人的帖子，更精准）
├── 浏览 Following tab：找出 <2h 的新帖
├── 过滤：话题相关 + 评论 <30 + views >5K
└── 记录 3-5 个候选

Step 2: For You tab 补充
├── 浏览 For You tab 找中型 KOL 新帖
├── 同样过滤条件
└── 追加候选列表

Step 3: 主动搜索高价值目标
├── 搜索格式：[关键词] min_faves:100（过滤出真正有互动的帖子）
├── 按"Latest"排序，找 <2h 的帖子，作者粉丝 10K-100K
└── 这个方法能找到 For You 没推给你的帖子

Step 4: 核心 KOL 开帖子通知（一次性设置）
├── 对 Profile 中的竞品账号 + 同领域 KOL 开 🔔 通知
├── 目标：帖子发出后 5-10 分钟内看到，进入真正的黄金窗口
└── 建议维护 10-15 个高价值 KOL 的通知列表

Step 5: 扫描热帖的顶部评论（高价值但常被忽略）
├── 找到互动量异常高的帖子（likes >500，评论 >50）
├── 查看该帖子的前 3-5 条高赞评论
├── 如果顶部评论本身有争议或可以被延伸 → 回复那条评论而不是原帖
└── 逻辑：顶部评论已经有自己的流量，在它下面回复竞争更低，且借了两层曝光
```

**Tier 1 评论质量标准：**

每条评论必须通过以下检查：
> 1. **截图测试**：把这条评论截图发出去，没有上下文，陌生人看完会不会觉得有价值？如果不会 → 重写
> 2. 这条评论如果去掉署名，别人能猜出是谁写的吗？（有独特性？）
> 3. 有没有让读者"哦，没想到"的内容？
> 4. 读完会不会想点你的主页看看你是谁？

---

### Tier 2 (500-2000) 评论策略：强化 niche 定位

**目标：成为某个细分话题下"必出现"的声音**

**选帖变化（Tier 2）：**

```
更严格的话题过滤：
✅ 只评论直接命中核心支柱的帖子（不再为了流量而评无关帖子）
✅ 可以挑战更大流量（views 10K-200K）
✅ 评论数窗口可以放宽到 <80

核心逻辑：Tier 2 不再需要借别人的流量建声誉，
而是需要"在这个 niche 里到处出现"来强化定位。
```

### 评论生成规则

**核心原则：评论是给所有看到这条帖子的人看的，不只是给原帖作者看的。**

写之前先问：**"这条评论，能让陌生人想点进我的主页吗？"**

**五种有效评论类型：**

**Type A: 增量价值（最高价值）**

本质：你知道一个别人不知道的细节，且这个细节真实存在。

```
[原帖论点的一个具体延伸或反面案例].
[一句话说明为什么会这样].
```

关键：**必须基于账号的真实经历或独家资产**，不能编造。
如果没有真实依据 → 选其他类型，不要强行用 Type A。

示例（真实感强）：
> "We saw this pattern with ~200 of our users. The ones who churn fastest aren't the ones who get stuck — they're the ones who succeed once and then can't replicate it. The first win is a trap."

示例（真实感差，不要这样写）：
> "We tested this at our company — found that X outperformed Y by 40%."（数字是编的，听起来假）

**Type B: 视角补充**

本质：原帖说了 A，你补充了原帖没提到的 B，两者都对。

```
[原帖视角] makes sense for [适用情况].
[你的补充视角] for [另一种情况].
The distinction matters because [原因].
```

**Type C: 善意反驳**

本质：你有具体理由认为原帖的结论在某些条件下不成立。

```
Counterpoint: [具体的不同结论].
[你的依据 — 必须具体，不能只是"我感觉"].
Might be a [情境/条件] difference.
```
注意：只在你真的有不同观点时用，不要为了显得深刻而强行反驳。

**Type D: 问题延伸**

本质：原帖触发了你脑子里一个更深的问题，你真的想知道答案。

```
This makes me wonder: [一个由原帖引发的真实问题].
[你自己的相关观察，表明你不是随便问的].
```

**Type E: Build in Public（真实挣扎）**

本质：原帖描述的情况你正在经历，你分享你的真实状态。

```
Going through this right now.
[具体情况 — 必须是真实的]. Still figuring out [挑战].
```
这种类型最容易写假，因为没有真实经历就会变得generic。如果没有真实相关经历 → 不要用。

**声音匹配检查（生成后必做）：**
- 句长是否匹配账号习惯？
- 有没有用账号的标志性表达？
- Emoji 使用是否一致？
- 语气是否匹配（干货型账号不写 "Going through this emotionally"）？

### 评论禁用词

- "Great post!" / "Love this!" / "So true!"
- "This!" / "Exactly this."
- "As a [身份], I think..."
- 纯 emoji 回复
- "DM me if you want more"
- 任何通用到任何人都能写的表达

### 输出格式

```markdown
# 💬 评论任务 | YYYY-MM-DD HH:MM
**账号**: @{handle} | **阶段**: {Tier 1/2} | **独家资产**: {一句话}

---

### 1. @{作者} — {话题落点：账号哪个支柱}
**帖子**: {摘要}
**数据**: 👁️{views} | ❤️{likes} | 💬{评论数} | ⏰{发布时间}
**链接**: https://x.com/...
**选中原因**: {为什么这条值得评论}

**草稿 A** ({类型}) — 依据：{用了什么独家资产}
> {评论内容}

**草稿 B** ({类型})
> {评论内容}

**声音检查**: ✅ 匹配 / ⚠️ {问题}

---
[更多帖子...]

## ⏳ 审批
回复序号执行（如: 1A, 2B）
回复 "改{序号}" + 反馈
```

## 🟢 Module 3: Post（发帖）

### Tier 策略

**Tier 1 (0-500)：帖子是补充，不是主力**

- 频率：1条/天，只在有真正值得说的内容时发
- 类型：Build in Public（真实故事/挣扎）> Data Drop（如果有数据）
- 原则：宁缺毋滥。一条烂帖子对账号的伤害大于不发
- 热点借势：只蹭你有独特视角的热点，不要为了蹭而蹭

**Tier 2 (500-2000)：建立 niche 内容壁垒**

- 频率：2条/天
- 类型：Niche 深度内容 + 偶尔热点借势
- 原则：成为这个 niche 里"最值得关注的账号"
- **同体量账号互扶**：主动 engage 粉丝数相近（±50%）的同领域账号。他们 follow-back 概率远高于大号，且会形成互相扩散的增长圈。每天花 10 分钟在 Profile 记录的"同体量账号"上评论/互动。

**Tier 2 内容日历（每周结构）**

Tier 2 阶段最大的陷阱是"随机发帖"——某天灵感来了发3条，然后消失5天。算法和读者都需要节奏感。

**Bookmark-bait vs Like-bait：每周都要有**

这是两种完全不同目的的内容，不能混用：

| 类型 | 目标 | 算法信号 | 内容特征 | 示例 |
|------|------|----------|----------|------|
| **Like-bait** | 情绪共鸣 | Like（1x）| 故事、挣扎、反直觉情绪 | "Worst week of my founder life." |
| **Bookmark-bait** | 参考价值 | Bookmark（10x）| 框架、数据、可复用判断 | "How to know if you've found PMF: [checklist]" |

每周至少各 1-2 条。只发 Like-bait，账号有温度但没有参考价值；只发 Bookmark-bait，账号有价值但读者不会产生情感连接。

推荐的每周内容节奏：

| 天 | 内容类型 | 目的 |
|----|----------|------|
| 周一 | Bookmark-bait：Data Drop / 框架 | 参考价值，算法 10x |
| 周二 | 评论日（不发帖） | 集中评论同 niche |
| 周三 | Like-bait：Story / Build in Public | 情感连接，引发互动 |
| 周四 | Bookmark-bait 或 Hot Take | 强化世界观 |
| 周五 | Like-bait：Question 帖 | 引发 reply，最强算法信号 |
| 周六 | 热点借势（如有）/ 休息 | 有好机会才发 |
| 周日 | 休息 | — |

**注意**：这是结构框架，不是强制内容。如果周三发生了重要的事，可以打破节奏。但没有特别情况时，按节奏走，保证每周至少 3-4 条高质量输出。

每月还应该有 1 条"内容总结帖"或"里程碑帖"，给关注你的人一个检查进度的机会。

**Tier 3 (2000+)：保证质量，系统化输出**

- 频率：2-3条/天
- 重心转向原创思想 + 热点借势
- 评论仍然重要，但是选择性的

---

### X 算法对帖子的影响（关键）

**理解这个才能写出高传播帖子：**

1. **前30分钟的互动率决定分发规模**
   - X 会先把你的帖子推给一小批人（你的粉丝中最活跃的）
   - 如果这批人快速点赞/回复 → 扩大分发
   - 如果没人互动 → 帖子基本死了
   - 实践意义：发帖时间选粉丝最活跃的时段，前30分钟尽量引导互动

2. **算法真实权重（X 开源代码实测）**
   - Retweet: 20x | Reply: 13.5x | Profile click: 12x | Link click: 11x | **Bookmark: 10x** | Like: 1x
   - Bookmark 权重远高于 Like，是严重被低估的信号
   - 实践意义：设计"值得收藏"的内容（数据、清单、可复用框架）能获得算法加成
   - 好的帖子要留一个"入口"让人回复（问题、争议点、不完整的故事）

3. **链接降低分发**
   - 带外链的帖子分发范围会被压制
   - 实践意义：如果要附链接，用回复而不是主帖

4. **热点 QT 的算法加成**
   - QT 热帖时，X 会把你的 QT 推给看过原帖的人
   - 但必须在热点 <4h 内发，之后加成迅速衰减

5. **媒体内容（图片/视频）加成**
   - 带图片的帖子分发范围通常 >纯文字
   - 实践意义：数据类内容用图表，故事类内容用截图/照片

6. **发帖后管理（发出去只是开始）**

   **A. 回复进来的评论（前 30 分钟最关键）**
   - 算法把"帖子有持续讨论"视为强分发信号
   - 回复每一条有实质内容的评论，哪怕只是一句话
   - 帖子 80% 的互动发生在前 3 小时，这段时间要保持在线

   **B. 主动在自己帖子下补充内容**
   这是很多增长实践者用的技巧，能显著延长帖子寿命：
   - 发帖 1-2 小时后，在自己帖子下回复一条补充信息
   - 格式："One thing I didn't mention: [额外的具体细节或例子]"
   - 或者："A few people asked about X — here's the full answer: [...]"
   - 效果：重新激活算法分发，让已经看过的人有新内容可以互动
   - 注意：补充内容必须真的有价值，不要只是为了刷存在感

   **C. 帖子起势时立刻延伸（momentum window）**
   - 信号：发帖 1 小时内，互动量明显高于该账号的平均水平
   - 动作：立刻发一条相关的新帖，或在原帖下补充一条"续集"
   - 原因：算法正在分发这个话题，跟进的内容会被推给同一批人
   - 延伸方向：原帖说了结论 → 延伸说过程；原帖说了失败 → 延伸说怎么解决
   - 时间窗口：互动高峰后 30-60 分钟内发出，超过 3 小时基本没有加成

### 高质量帖子生成流程

```
Step 1: 选题（必须先确认"凭什么只有我能说这个"）
├── 今日热点 × 账号独特视角（有时间窗口，优先处理）
├── 账号独家资产（真实经历/数据/观察）
├── Niche 内的深度洞察
└── 如果无法回答"只有我能说这个因为..."→ 不发

Step 1.5: Lean Writing 验证（可选但推荐）
├── 先把核心观点写成一条单行 hot take（<100字符）
├── 如果这个单行帖子本身就有力 → 可以直接发，或扩展成更长内容
├── 如果单行写不出来 → 说明观点还不够清晰，先打磨观点再写帖子
└── 用读者反馈决定是否值得写成 thread（有互动再扩展）

Step 2: 格式决策
├── Hot take / 观点 → 纯文字，<100字符
├── 数据/洞察 → 纯文字或配图表，<220字符
├── 故事 → 纯文字，可以稍长
├── 热点借势 → QT 原帖，评论 2-4 句
└── Thread → 首帖要能独立成立，每条 <200字符

Step 3: 首行（Hook）
必须用以下之一：
1. 数字开头："47% of..."
2. 反直觉："Everyone says X. The data shows Y."
3. 情绪："Worst week of my startup life."
4. 问题："Why do 90% of AI tools feel the same?"
5. 大胆声明："Cold email is dead."
6. 故事开头："A customer just told me something wild."

数字使用原则：用精确数字，不用圆整数字
→ "12,847 个用户" 比 "约 13,000 个用户" 更有说服力
→ 原因：精确数字暗示真实经历，圆整数字暗示估算或编造
→ "We talked to 23 founders" 比 "We talked to many founders" 好得多

Step 4: 世界观一致性检查
├── 对照 Profile 中提取的世界观
├── 问：这条帖子在强化世界观，还是游离在外？
└── 游离在外不等于不能发，但要有意识地做，不要随机漂移

Step 5: 声音匹配
对照账号声音指纹检查：句长、语气、emoji、标志性表达

Step 6: 发帖前检查清单（见下）
```

### 发帖前检查清单

- [ ] 第一行能让陌生人想看下去？（独立测试：只看第一行）
- [ ] 有具体数据或真实故事？（不是通用观点）
- [ ] 数字是精确的而不是圆整的？（"2,347" 而不是 "约 2,000"）
- [ ] 这个内容只有这个账号能说？为什么？
- [ ] 没有禁用词？声音匹配这个账号？
- [ ] 如果要加链接，放在回复里而不是主帖？
- [ ] 有没有给人留一个回复的入口？（问题/争议点/待续故事）
- [ ] 这条帖子是 Bookmark-bait 还是 Like-bait？本周是否两类都有？
- [ ] 这条帖子在强化账号的世界观，还是随机游离？
- [ ] 这条帖子有没有让人"值得收藏"的价值？（bookmark = 10x 算法权重）
- [ ] 发完后 30 分钟内回复进来的评论？（延长帖子寿命）

### 内容类型与阶段匹配

| 类型 | Tier 1 | Tier 2 | 说明 |
|------|--------|--------|------|
| **QT 热点** | ⚠️ 仅有独特视角时 | ✅ 常规武器 | 热点 <4h 才有算法加成 |
| **Build in Public** | ✅ 主力内容 | ✅ 保持真实感 | 必须是真实故事，不能编 |
| **Data Drop** | ✅ 如有数据 | ✅ 核心内容 | 具体数字 > 百分比 > 感受 |

| **Contrarian** | ⚠️ 需谨慎（容易翻车） | ✅ 成长后更有效 | 必须有具体依据 |
| **Thread** | ❌ 太早（没有受众） | ✅ 5-7条为佳 | 用 Lean Writing 验证后再写 |
| **Question** | ✅ 低成本互动 | ✅ 增加 reply | 不要天天问问题 |

## 执行流程

### 首次使用

```
1. Module 0 — 生成账号画像（存 memory）
2. Module 1 — 今日热点机会
3. Module 2 — 评论目标（Tier 1 重点）
4. Module 3 — 发帖草稿（Tier 1 少量）
```

### 日常（画像已有）

```
"跑今日日报" / "早报" / "daily" / "跑 X 运营"
→ 全流程：Module 1（三层扫描）→ Module 2（评论候选）→ Module 3（发帖草稿）
→ 打包成一份 Signal Pack 输出，一次审批

"找评论机会" / "找帖子评论"
→ 只跑 Module 2（评论目标 + 草稿）

"帮我写推" / "帮我写帖子"
→ 只跑 Module 3（发帖草稿）

"看热点" / "今天有什么热点"
→ 只跑 Module 1 Layer A + B（不拉账号数据）

"刷新画像" / "更新账号信息"
→ 重跑 Module 0
```

### 边缘情况

```
新号（0条帖子）:
→ 跳过声音指纹和历史分析
→ 询问用户：目标受众、擅长领域、有哪些真实经历可用
→ 用回答填充 Profile

话题超出内容支柱:
→ 提醒："这个话题不在核心领域，评论/发帖可能显得不真实"
→ 询问是否继续
```
